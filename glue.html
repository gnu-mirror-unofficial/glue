GNU GLUE - Links Users Everywhere

With GNU Groupware, you can make a difference.

The purpose of the GNU GLUE project is to create a distributed groupware application framework based on emerging new Internet standards, such as XML, WebDAV and RTSP, suitable for both synchronous and asynchronous and both on-line and disconnected operation.

Tasks:

  • evaluate available software and design options
  • create project-related "special interest-groups" (SIGs)
  • design extensible and modular core groupware framework
  • develop groupware server(s) and other back-end components
  • develop groupware client(s) and helper applications
  • develop application programming framework
  • provide migration/interoperability support for legacy groupware
  • develop a set of advanced groupware applications based on the groupware framework
  • write documentation (manuals, FAQs, tutorials)
  • experiment with new groupware design and implementation approaches
  • create world-wide user/developer support infrastructure
  • lobby for regional government support of project

As the GNU GLUE project will build on a lot of existing free software packages, it will have to inherit the terms of the respective licenses. However, it is intended that most of the system will be published under terms that are compatible with the GNU General Public License. Support for legacy systems migration and interoperability (such as for Lotus Notes, proprietary SQL databases such as Oracle or MySQL, and Microsoft Windows platform support) may rely on availability of the respective proprietary software packages, but will not be required to use GNU GLUE.

Project roadmap:

  • Create feature wish list and evaluate options: until end of 1 Q 2000
  • Draft overall architecture and create series of demonstration prototypes (0.x): until 3 Q 2000
  • Usable bootstrapping version (1.x): 4 Q 2000
  • Stabilized release (version 2.x) with complete reference and tutorial documentation and user-supported implementation workshop templates: 4 Q 2001
